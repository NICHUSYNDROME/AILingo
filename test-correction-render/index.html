🧪 纠正渲染测试

对比不同渲染策略的效果，选择最优方案

📌 测试用例 1：おすすめラーメンは何です？ → おすすめのラーメンは何ですか？

原始文本
おすすめラーメンは何です？
策略 A — 当前 diff 算法
直接运行现有的 findDifferences + generateAnnotatedMessage
策略 B — 只高亮新增字符（用户期望）
在 corrected 文本中找到相对于 original 新增的字符，只高亮新增部分
策略 C — 删除线 + 新增绿色
被替换的原词标删除线，新增字符标绿色
策略 D — 基于 changes 数组逐条标注
直接解析 2A 的 changes 数组，逐条渲染修正
